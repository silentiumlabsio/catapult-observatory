# Privacy and Responsible Use

## Local-First Design

Catapult Observatory is designed to operate locally.

No cloud backend is required for its core functions.

## Data the Platform May Store

Depending on the enabled features, the platform may store:

- Wi-Fi station counters
- DHCP lease information
- Local connection metadata
- Bluetooth observations
- Performance measurements
- Sensing sessions
- Assessment notes
- Evidence hashes
- User-approved exports

## Data the Platform Is Not Designed to Collect

The platform is not designed to collect:

- Passwords
- Credentials
- Decrypted HTTPS content
- Private messages
- Transferred files
- Application payload content

## Public Screenshots

Before publishing screenshots, remove or blur:

- MAC addresses
- Private IP addresses where appropriate
- Personal device names
- Bluetooth identities
- Private DNS destinations
- Local hostnames
- Usernames
- Internal file paths
- Session identifiers

## Authorized Use

Use Catapult Observatory only with:

- Hardware you own
- Devices you own
- Networks you control
- Systems you have explicit permission to assess

## Sensing Ethics

RSSI-based sensing should be demonstrated transparently.

The project should not be used to secretly monitor people or make unsupported claims about identity, body pose, or exact location.

## Evidence Handling

Assessment evidence should be:

- Minimized
- Sanitized
- Stored locally
- Access-controlled
- Deleted when no longer required
- Shared only with authorized recipients
